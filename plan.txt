we fetch Vagaro's support docs from here https://support.vagaro.com/api/v2/help_center/en-us/articles. (fetch.py)

chunk em with with the following philosophy:  (chunk.py)
    Short articles (≤ 1500 tokens) → single chunk, done
    Longer articles → HTML is parsed, split at every H2/H3 heading into sections
    Sections are greedily merged — consecutive sections get combined until adding the next one would exceed 1500 tokens, then a new chunk starts
    Tables, iframes, code blocks → always their own standalone chunk, never merged
    Oversized sections (a single section > 1500 tokens) → sliding token windows of 1500 with 150-token overlap

embed_and_store with (embed_and_store.py)
    embed and store chunks with openai small embedding into a local chroma db
